# Requerimientos Funcionales

## Funcionalidad General

- [ ] El programa debe iniciar (inicialmente .\_.)
- [ ] El programa debe mostrar una interfaz gráfica al usuario.

## Atajos de Teclado

- [ ] El programa debe mostrar atajos de teclado para:
  - [ ] Abrir archivo
  - [ ] Guardar archivo
  - [ ] Salir del programa
  - [ ] Ir a la sección "Acerca de"
  - [ ] Ir a la sección de licencias
  - [ ] Cambiar tema (Opcional)
  - [ ] Cambiar idioma

## Funcionalidad del Menú

- [ ] El programa debe tener un menú completamente funcional con las siguientes opciones:
  - Archivo
    - [ ] Abrir archivo
    - [ ] Guardar archivo
    - [ ] Salir del programa
  - Ajustes
    - [ ] Cambiar tema (Opcional)
    - [ ] Cambiar idioma
  - Ayuda
    - [ ] Acerca de
    - [ ] Licencias

## Funcionalidad de la Barra Lateral

- [ ] El programa debe tener una barra lateral con las siguientes secciones:
  - [ ] Abrir archivo
  - [ ] Cambiar idioma

## Manejo de Archivos

- [ ] El programa debe permitir al usuario abrir un archivo mediante arrastrar y soltar.
- [ ] El programa debe permitir al usuario abrir un archivo mediante el menú.
- [ ] El programa debe permitir al usuario abrir un archivo mediante un atajo de teclado.
- [ ] El programa debe permitir al usuario abrir un archivo mediante la barra lateral.
- [ ] El programa debe mostrar el contenido del archivo abierto en un editor de texto (Monaco Editor o similar).
- [ ] El programa debe permitir al usuario editar el contenido del archivo abierto en el editor de texto y ver los cambios reflejados en tiempo real.
- [ ] El programa debe permitir al usuario guardar el contenido del archivo abierto mediante el menú.
- [ ] El programa debe permitir al usuario guardar el contenido del archivo abierto mediante un atajo de teclado.
- [ ] El programa debe permitir al usuario guardar el contenido del archivo abierto mediante un botón dentro del editor de texto.

## Funcionalidades Adicionales

- [ ] El programa debe mostrar una sección "Acerca de" con información del programa y enlaces a redes sociales.
- [ ] El programa debe soportar múltiples idiomas (i18n).
- [ ] El programa debe ser fluido y responsivo, sin congelarse o bloquearse durante la conversión de archivos.
- [ ] El programa debe mostrar un diálogo para las licencias de terceros utilizadas en el programa.
- [ ] El programa debe contar con un sistema robusto de manejo de errores.
- [ ] El sistema debe ser intuitivo y fácil de usar para usuarios de todos los niveles de experiencia.

## Compatibilidad y Rendimiento

- [ ] El programa debe ser compatible con los sistemas operativos más comunes (Windows, macOS, Linux).
- [ ] El programa debe ser ligero y no consumir demasiados recursos del sistema.
- [ ] El programa debe ser compilado en un binario ejecutable para facilitar su distribución y uso por parte de los usuarios finales.
- [ ] El binario del programa debe ser ligero.

## Manejo de Tipos de Archivo

- [ ] El programa debe manejar los tipos de archivo admitidos (incluso cuando se use el sistema de arrastrar y soltar).
