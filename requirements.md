# Functional Requirements

## General Functionality

- [ ] The program must start (initially .\_.)
- [ ] The program must display a graphical user interface to the user.

## Keyboard Shortcuts

- [ ] The program must display keyboard shortcuts for:
  - [ ] Open file
  - [ ] Save file
  - [ ] Exit the program
  - [ ] Go to the about section
  - [ ] Go to the licences section
  - [ ] Change theme (Optional)
  - [ ] Change language

## Menu Functionality

- [ ] The program must have a fully functional menu with the following options:
  - File
    - [ ] Open file
    - [ ] Save file
    - [ ] Exit the program
  - Settings
    - [ ] Change theme (Optional)
    - [ ] Change language
  - Help
    - [ ] About
    - [ ] Licences

## Sidebar Functionality

- [ ] The program must have a sidebar with the following sections:
  - [ ] Open file
  - [ ] Change language

## File Handling

- [ ] The program must allow the user to open a file via drag and drop.
- [ ] The program must allow the user to open a file via the menu.
- [ ] The program must allow the user to open a file via a keyboard shortcut.
- [ ] The program must allow the user to open a file via the sidebar.
- [ ] The program must display the content of the opened file in a text editor (Monaco Editor or similar).
- [ ] The program must allow the user to edit the content of the opened file in the text editor and see the changes reflected in real time.
- [ ] The program must allow the user to save the content of the opened file via the menu.
- [ ] The program must allow the user to save the content of the opened file via a keyboard shortcut.
- [ ] The program must allow the user to save the content of the opened file via a button inside the text editor.

## Additional Features

- [ ] The program must display an "About" section with program information and links to social media.
- [ ] The program must support multiple languages (i18n).
- [ ] The program must be fluid and responsive, without freezing or blocking during file conversion.
- [ ] The program must display a dialogue for third-party licences used in the program.
- [ ] The program must have a robust error handling system.
- [ ] The system must be intuitive and easy to use for users of all experience levels.

## Compatibility and Performance

- [ ] The program must be compatible with the most common operating systems (Windows, macOS, Linux).
- [ ] The program must be lightweight and not consume excessive system resources.
- [ ] The program must be compiled into an executable binary for easy distribution and use by end users.
- [ ] The program binary must be lightweight.

## File Type Management

- [ ] The program must handle supported file types (even when using the drag and drop system).
